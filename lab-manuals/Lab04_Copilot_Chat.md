# 演習 4 - Microsoft 365 Copilot Chat で自治体向け提案準備を行う

> 所要時間: 20 分

---

## シナリオ

あなたは空間情報・測量分野の企業に所属する営業・技術営業担当者です。サンプル自治体「自治体A」から、老朽化する橋梁・道路・河川施設の点検効率化と、豪雨災害時の状況把握を高度化したいという相談を受けています。

この演習では、Copilot Chat を使って複数の資料を読み取り、自治体向けの提案仮説、関係者別の関心事項、次回ヒアリング項目を作成します。空間情報、測量、衛星データ、3D都市モデル、GIS活用をどう提案につなげるかを考える演習です。

## 使用するファイル

- `自治体A_案件概要.docx`
- `空間情報DX提案パッケージ_概要.docx`
- `ヒアリングメモ_自治体A.docx`

## この演習で学ぶこと

- Copilot Chat で複数ファイルを参照する方法
- Goal、Context、Source、Expectation を含むプロンプトの作り方
- 自治体・インフラ管理向けの提案仮説を整理する方法
- 根拠がある内容と推測を分ける確認方法

---

## 1. Copilot Chat を起動する

1. [Microsoft 365 ポータル](https://m365.cloud.microsoft/) から **Copilot Chat** を開きます。
2. 画面上部で  **Work IQ（もしくは作業モード）** が有効になっていることを確認します。Work IQが無効もしくはWeb モードになっている場合は切り替えます。
3. 新しいチャットを開始します。

---

## 2. 基本プロンプトで案件を要約する

1. 入力欄で **+** または **/** を使い、使用する 3 ファイルを追加します。
   - `自治体A_案件概要.docx`
   - `空間情報DX提案パッケージ_概要.docx`
   - `ヒアリングメモ_自治体A.docx`
2. 次のプロンプトを入力します。

```prompt
添付した資料をもとに、自治体A向けの空間情報DX提案に関する案件概要を要約してください。
```

3. 出力を確認し、情報がどの程度具体的に整理されたかを確認します。

---

## 3. 4要素を入れたプロンプトに改善する

単純なプロンプトでの要約依頼では、求めている結果が得られない場合があります。要約に必要な視点や背景情報など、切り口として使用できる情報をプロンプトに含めて改めて要約させます。

```prompt
あなたは空間情報・測量分野の企業に所属する自治体向けアカウント担当者です。
添付した3つの資料をもとに、自治体A向けの初回提案準備用アカウントプランを作成してください。

目的:
- 顧客の課題、意思決定者、導入に向けた懸念を整理する
- 空間情報、航空測量、衛星データ、GIS、3D都市モデルを使った価値仮説を作る
- 次回ヒアリングで確認すべき質問を準備する

出力形式:
1. 顧客の現状と課題
2. 提案すべき価値仮説
3. 関係者別の関心事項
4. 想定される反論と切り返し
5. 次回ヒアリングで確認する質問10個
6. 直近2週間のアクションプラン
```

---

## 4. 不足情報を洗い出す

AIとの対話の中で、不足している情報を補足させます。人間が見落としている観点や異なる視点からの気づきを得られる場合があります。

```prompt
先ほどのアカウントプランについて、提案前に不足している情報を洗い出してください。
不足情報は「確認すべき理由」「確認相手」「優先度」の列を持つ表にしてください。
```

---

## 5. 関係者別に質問を作る

ヒアリングやプレゼンテーションにおける双方の質問内容について、あらかじめ洗い出す作業をAIに依頼してスムーズなやり取りができるよう備えます。

```prompt
不足情報を確認するためのヒアリング質問を、防災担当、道路管理担当、情報政策担当、財政担当の4者に分けて作成してください。
質問は相手が答えやすい自然な日本語にし、各質問の意図も添えてください。
```

---

## 6. 生成結果を検証する

AIが出力した内容や人間が想定している内容が、実態に即しているかをチェックさせることで正確性の向上を目指します。

```prompt
今の回答のうち、添付資料に根拠がある内容と、推測が含まれる内容を分けてください。推測には「要確認」と明記してください。
```

---

## 7. 完成物を作る

```prompt
ここまでの内容を、社内の提案検討会で共有するためのA4 1ページ相当の内容にまとめてください。
見出しは「案件状況」「提案仮説」「確認事項」「リスク」「社内依頼事項」としてください。
```

---

## 実務で使えるプロンプトサンプル

```prompt
添付した会議メモと提案資料をもとに、次回商談で確認すべき論点を「顧客課題」「意思決定条件」「技術確認」「契約・予算確認」に分けて整理してください。
```

```prompt
この案件について、受注確度を高めるために今週実施すべきアクションを5つ提案してください。各アクションには、目的、担当候補、想定アウトプットを含めてください。
```

```prompt
添付資料の内容から、顧客にまだ確認できていない前提条件を洗い出してください。推測を含む項目には「要確認」と明記してください。
```

## 完了

この演習では、Copilot Chat を使って空間情報DX案件の提案準備を行いました。
