# 演習 5 - Copilot Studio で空間情報ナレッジ エージェントを作成する

> 所要時間: 35 分

---

## シナリオ

提案準備のたびに、GIS、航空測量、衛星データ、3D都市モデル、インフラ管理、防災活用に関する説明文や注意点を探すのに時間がかかっています。Copilot Studio を使って、承認済みの提案ナレッジだけを参照する「空間情報提案アシスタント」を作成します。

## 使用するファイル

- `空間情報ナレッジ_FAQ.docx`
- `空間情報ナレッジ_提案ポリシー.docx`
- `空間情報ナレッジ_反論対応集.docx`
- `空間情報ナレッジ_実証ステップ.docx`

## この演習で学ぶこと

- 自然言語でエージェントを作成する方法
- ナレッジソースを限定する方法
- エージェント指示を改善する方法
- 推奨プロンプトを設定し、回答品質をテストする方法

---

## 1. 新しいエージェントを作成する

1. [Microsoft 365 ポータル](https://m365.cloud.microsoft/) 左側のメニューペインに表示される **新しいエージェント** または Copilot Studio のエージェント作成画面を開きます。
   - エージェント作成画面が表示されない場合は、作成権限がない可能性があります。その場合は講師の環境でデモとして実施します。
2. エージェント作成用のチャット(Agent Builder)に次のプロンプトを入力します。

```prompt
「空間情報提案アシスタント」という名前のエージェントを作成してください。
このエージェントは、空間情報・測量分野の企業に所属する営業・技術営業担当者が自治体やインフラ管理者向けの提案を準備する際に、承認済みの提案ナレッジをもとに質問へ回答するためのものです。
回答には必ず根拠となるナレッジソースの内容を使い、資料にない内容は推測しないでください。
```

---

## 2. エージェントの説明と指示を確認する

1. 画面右側で **構成** タブを開きます。
2. 名前、説明、指示が意図どおりになっているか確認します。
3. 指示が抽象的な場合は、画面左側下部に表示されている入力欄に次のプロンプトを入力して改善します。

```prompt
エージェントの指示を次の方針で更新してください。

- 営業・技術営業担当者向けに、簡潔で実務的に回答する
- 回答は「結論」「理由」「顧客へ説明する言い方」「注意点」の順にする
- 費用、契約、保証、個人情報、災害時運用に関する質問では、資料に書かれた範囲を超えて断定しない
- 個別見積や法務判断が必要な場合は、担当部門への確認を促す
- 顧客へそのまま送れる文章が必要な場合は、丁寧な社外向け文面にする
```

---

## 3. ナレッジソースを追加する

1. 画面右側の構成で **ナレッジ** セクションへ移動します。
2. **すべての Web サイトを検索します** に相当する設定が有効な場合は無効にします。
3. 使用する4つのWord 文書を追加します。
   - `空間情報ナレッジ_FAQ.docx`
   - `空間情報ナレッジ_提案ポリシー.docx`
   - `空間情報ナレッジ_反論対応集.docx`
   - `空間情報ナレッジ_実証ステップ.docx`
4. ナレッジの取り込みが完了するまで待ちます。

---

## 4. 推奨プロンプトを作成する

推奨プロンプトはユーザーに対してエージェントへの質問内容を提案する役割を果たします。最初の質問ステップを踏み出すための手助けとなります。

```prompt
この空間情報提案アシスタントで使う推奨プロンプトを5つ作成してください。
自治体向け提案、技術説明、反論対応、実証計画作成に使いやすいものにしてください。
```

さらに、手動で以下の推奨プロンプトを追加します。画面右側の構成の下部にある **推奨プロンプト** セクションで **+ 推奨プロンプトを追加する** から実行します。

| タイトル | メッセージ |
| --- | --- |
| 初回提案の価値訴求 | 防災危機管理課向けに、空間情報DXの価値を3点で説明してください。 |
| 反論対応 | 顧客から「既存GISと連携できるか不安」と言われた場合の回答例を作ってください。 |
| 実証範囲確認 | 90日実証で最初に対象にすべき施設やデータを整理してください。 |
| 個人情報の注意点 | 空間情報を扱う提案で、個人情報や公開情報の扱いについて注意点を整理してください。 |
| 庁内説明 | 財政課向けに費用対効果を説明する短い文章を作ってください。 |

---

## 5. 回答品質をテストする

**試してみる** 画面で、次の質問を入力します。

```prompt
自治体Aの防災危機管理課に、初回提案で伝えるべき価値を3点で説明してください。
```

回答を確認し、以下の観点で評価します。

- ナレッジソースに基づいているか
- 自治体向けに使える表現になっているか
- 不明点を推測で断定していないか
- 次の提案アクションにつながるか

---

## 6. 失敗しやすい質問でテストする

```prompt
今回の案件の最終見積金額を教えてください。
```

期待される動作は、資料にない金額を推測せず、個別見積が必要であることを伝えることです。期待どおりでない場合は、指示を追加します。

```prompt
資料に明記されていない金額、契約条件、保証内容は推測で回答しないよう、指示を強化してください。
```

---

## 7. エージェントを作成する

実際の業務では、エージェントを構成した後は作成（保存）して共有することで、組織内の別のユーザーにも同じエージェントを活用してもらうことが可能となります。今回は演習となるため、 **作成はしないでください** 。

1. 指示、ナレッジ、推奨プロンプト、テスト結果を確認します。
2. 問題がなければ 作成 を選択します。
3. 作成後、必要に応じてチーム内で共有します。

---

## 実務で使えるプロンプトサンプル

```prompt
このエージェントの回答品質を確認するためのテスト質問を10個作成してください。通常質問、曖昧な質問、資料にない質問を混ぜてください。
```

```prompt
エージェントの指示を改善してください。資料にないことを推測しない、回答に根拠を示す、顧客向け表現に変換する、という3点を強化してください。
```

```prompt
登録済みナレッジをもとに、営業担当者が初回提案前に使う推奨プロンプトを5つ作成してください。タイトルと本文を分けてください。
```

## 完了

この演習では、Copilot Studio を使って、空間情報DX提案で使えるナレッジ エージェントを作成しました。
